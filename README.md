# Работа с git

----

### Установка git через консоль/терминал на MacOS

1. Запусть терминал
2. Выпонить команду /usr/bin/git
3. Нажать Install в модальном окне
4. Для проверки успешной установки в терминале ввести команду git version

### Настройка git

1. Выполнить команду git config --global user.name "UserName", в кавычках вместо UserName узакывается имя пользователя латиницей
2. Выполнить команду git config --global user.email "user@mail.ru", в кавычках вместо user@mail.ru узакывается электронный адрес пользователя
3. Выполнить команду git config --list для проверки установленных значений

### Инициализация репозитория

1. Создать папку для будущего репозитория mkdir project, где project - названи создаваемой папки, при неодходимости указывать расположение папки
2. Перейти в папку созданную на шаге 1 cd project/
3. Выполнить команду инициализации git init
4. Проверить статус репозитория git status

### Заполнение репозитория данными

1. Создать файл в папке репозитория touch file-name.txt
2. Добавить файл в git для контроля версий git add --all или git add file-name.txt
3. Создать коммит git commit -m 'message with change description'
4. Проверить историю коммитов git log

### Создание публичного репозитория

1. Открыть страницу github.com
2. Нажать на кнопку Sung Up
3. Пройти регистрацию
4. Открыть страницу профиля
5. Перейти на вкладку repositories
6. Нажать на кнопку New
7. Ввести название репозитория в поле Repository name
8. Нажать на кнопку Create repository

### Создание ssh ключей и довавление их в GitHub

1. Открыть терминал
2. Выполнить команду ssh-keygen -t ed25519 -C "user@mail.ru", где в кавычках email указанный при регистрации на GitHub
3. Нажимать кнопку Enter до появления картинки
4. Скопировать в буфер содержимое файла с *публичным* ключем id-ed25519.pub pbcopy < ~/.ssh/id-ed25519.pub
5. Перейти в GitHub
6. Открыть настройки профиля (settings)
7. Открыть вкладку SSH and GPG keys
8. В разделе SSH keys нажать на кнопку New SSH key
9. Заполнить поле Title названием ключа
10. В поле Key type выбрать значение Authentication key
11. В поле Key вставить значение ключа из буффера
12. Нажать на кнопку Add SSH key
13. Перейти в терминал
14. Выполнить команду ssh -T git@github.com
15. Ввести yes

### Привязка локального репозитория к удаленному

1. Перейти на странуцу удаленного репозитория
2. Выбрать SSH
3. Скопировать ссылку
4. Перейти в терминал
5. Перейти в папку с репозиторием cd project/
6. Выполнить команду на подлючение удаленного репозитория git remote add origin repository-link, где repository-link - скопированная ссылка
7. Проверить подключение git remote -v

### Отправка изменений в удаленный репозиторий

1. Выполнить в терминале команду git push -u origin main (вместо main нужно указывать master для старых репозиториев)
2. Перейти в репозитьрий в GitHub
3. Проверить на вкладке Code отображение данных из локального репозитория

**при последующих отправках изменений в удаленных репозиторий используется команда git push без дополнительных параметров**
 
#### Хеш данных

Хеш - идентификатор коммита в Git.  
Хеширование - преобразование набора данных для получения их "отпечатка".  
Хеш коммита получается с помощью алгоритма SHA-1.  
Хеш состоит из цифр 0-9 и латинских букв A-F.  
Свойства хеша: если дважды получить хеш для одного набора данных, результат будет одинаковым.  
Если изменить хоть что-то в исходных данных, хеш изменится.  
Хеш - основной идентификатор коммита, позволяет узнать автора, дату и содержимое закоммиченных файлов.  
Хеши и таблицу хеш → информация о коммите Git сохраняет в служебных файлах в папке .git.  

#### Работа с git log

Лог содержит описание коммита: хеш, автор, дата, сообщение.  
Сокращенный лог помогает быстро найти нужный коммит среди множества.  
Команда git log --oneline выводит сокращенный лог с хешами и комментариями.  
Уникальная длина сокращенных хешей помогает идентифицировать коммит.  

#### Head

Файл HEAD находится в папке .git и указывает на последний коммит в системе git.  
Для проверки содержимого файла HEAD можно использовать команду cat HEAD.  
Внутри файла HEAD находится ссылка на служебный файл refs/heads/master, содержащий хеш последнего коммита.  
При работе с Git можно заменить хеш на слово HEAD для передачи последнего коммита.
